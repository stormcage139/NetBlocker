# NetworkBlocker

Простой инструмент для блокировки сетевого трафика процессов в Windows и Linux через правила firewall.

## 🚀 Быстрый старт (Windows)

### 0) Установить `uv` (если ещё не установлен)
Windows
```powershell
powershell -c "irm https://astral.sh/uv/install.ps1 | more"
```
Или для Linux(curl)
```bash
curl -LsSf https://astral.sh/uv/install.sh | less
```


> ✅ `uv` — это удобный менеджер Python-проектов: он может создать виртуальное окружение, синхронизировать зависимости и запускать скрипты.


### 1) Синхронизировать проект (установить зависимости)
```powershell
uv sync
```

> 🔎 Команда `uv sync` прочитает `pyproject.toml` и установит `psutil`, `PySide6` и другие зависимости.

### 2) Запустить приложение (рекомендуемый вариант)
```powershell
uv run .\app.py
```

---

## 🐧 Быстрый старт (Linux)

### 0) Установить `uv` (если ещё не установлен)
```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

### 1) Синхронизировать проект (установить зависимости)
```bash
uv sync
```

### 2) Запустить приложение (требуется root для работы с iptables)
```bash
sudo uv run ./app.py
```

> ⚠️ На Linux приложение использует `iptables` для блокировки трафика по UID пользователя процесса. Запуск от root обязателен. Блокировка применяется ко всем процессам пользователя.

---

## 🧠 Как работает

- Список процессов пользователя собирается через `psutil`.
- Для каждого процесса проверяется наличие правила `Block_<имя_процесса>` в Windows Firewall.
- Кнопка **Toggle** создает/удаляет правило, блокируя или разрешая исходящий трафик для выбранного процесса.

---

## 🛠 Технические детали

- GUI: PySide6 (Qt)
- Windows: Логика работы с firewall через `netsh`
- Linux: Логика работы с firewall через `iptables` (блокировка по UID)

---

## ⚠️ Требование

Запуск должен быть от имени администратора (Windows) или root (Linux), иначе команды firewall не выполняются.
